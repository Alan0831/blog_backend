# token 登录态前端配合事项

## 背景

后端 token 有效期已统一调整为 24 小时。需要登录态的接口如果 token 缺失、无效或过期，会返回 HTTP 401，并在响应体中给出明确错误码，前端收到后需要清理本地登录态并引导用户重新登录。

## 登录响应新增字段

登录成功响应的 `data` 中除原有 `token` 外，新增：

```json
{
  "tokenExpiresIn": "24h",
  "tokenExpiresAt": 1781673600000
}
```

- `tokenExpiresIn`: token 有效期说明，目前固定为 `24h`。
- `tokenExpiresAt`: token 到期时间戳，单位毫秒。

前端保存 token 时建议同步保存 `tokenExpiresAt`，进入页面或发起登录态接口前可先判断是否已超时。

## 请求头要求

调用需要登录的接口时，请在请求头中携带：

```http
Authorization: Bearer <token>
```

本次后端兼容了 `token`、`x-token`、`x-access-token` 等旧 header，但推荐统一迁移到 `Authorization: Bearer <token>`。

## 401 响应处理

后端返回示例：

```json
{
  "status": "error",
  "data": {
    "errorType": "tokenExpired",
    "expiredAt": 1781673600000
  },
  "errorCode": "TOKEN_EXPIRED",
  "errorMessage": "登录已过期，请重新登录"
}
```

前端需要重点处理以下错误码：

1. `TOKEN_MISSING`: 未携带 token，提示用户登录。
2. `TOKEN_EXPIRED`: token 已超过 24 小时，清理本地用户信息和 token，跳转登录。
3. `TOKEN_INVALID`: token 格式错误或校验失败，清理本地用户信息和 token，跳转登录。

涉及接口包括但不限于 `/createVideoComment`、`/createComment`、`/deleteVideoComment`、`/deleteComment`、`/updateUser` 等需要登录的接口。
